## 1. 專案是什麼

WAFERLOCK（電子鎖品牌）的售後服務體系，由數個獨立的網頁 / 服務模組組成，共用同一個 Supabase 後端：

- **客戶保固登錄**（WordPress 外掛 + 保固卡）
- **CRM 客服後台**（`waferlock_crm.html`，本次主要施作對象）
- **LINE 客服 / 客戶端**（`waferlock_LINE.html`）
- **師傅端**（`waferlock_tech.html`）
- **LIFF 會員綁定**（`liff-bind.html`）
- **Supabase Edge Functions**（LINE 推播 / webhook）

各模組是獨立的靜態 HTML（無打包工具），靠 Supabase 資料表 + Realtime 串接，不是單一 SPA。

## 3. CRM 後台 `waferlock_crm.html`（已深入檢視）

### 3.1 技術棧

- 純單一 HTML 檔，**無建置工具**。瀏覽器內以 CDN 載入：
  - React 18.2.0 + ReactDOM（development build，UMD）
  - Babel Standalone 7.23.5（瀏覽器即時編譯 JSX）
  - `@supabase/supabase-js@2`
  - Google Fonts：Noto Sans TC、JetBrains Mono
- 樣式：檔頭 `<style>` 內的 CSS 變數主題（`--brand:#0F2744`、`--accent:#38BDF8` 等）。
- 狀態管理：單一 `App` 元件 + React Context（`Ctx`），無 Redux 等。

### 3.2 Supabase 連線（硬編碼於檔內）

- URL：`https://nkyanyjgfrmovjoqevro.supabase.co`
- Key：**anon public key**（前端用、可公開，但代表所有資料表 RLS policy 目前皆 `using(true)` 全開）
- 透過 `sb.channel(...).on('postgres_changes',...)` 做跨裝置 **Realtime 即時同步**。

### 3.3 使用到的資料表（`.from()` 實際出現）

`customers`、`repairs`、`complaints`、`installs`、`conversations`、`workers`、`bookings`、`config`、`crm_users`

- 程式內以 camelCase 物件操作，透過 `xxxToDb` / `dbToXxx` 函式與資料庫 snake_case 欄位互轉。
- 本機狀態 `db` 形狀：`{customers:[], repairs:{}, complaints:{}, installs:{}, conversations:[], workers:[], bookings:[], wfCounter, version}`。

## 4. 使用者登入系統（本次完成的改動）

對應 SQL：[`supabase_users_history.sql`](supabase_users_history.sql)（已於正式 Supabase 執行）。

### 4.1 資料庫

- **`crm_users` 表**：`id, name, account, pin, role, active, created_at`
  - `role` = `'admin'` 或部門名稱
  - 預設 4 組帳號，**密碼皆 `1234`**：
    - `admin`（系統管理員 / admin）
    - `meimei`（客服-小美 / 客服課）
    - `aming`（客服-阿明 / 客服課）
    - `ache`（業務-阿哲 / 內銷課）
- **`repairs` / `installs`** 新增 `history jsonb default '[]'`：每筆為 `{ts, by, action}`。
- `crm_users` 加入 realtime publication。

### 4.2 前端行為

- 未登入 → 顯示 `LoginScreen`；登入狀態存於 `localStorage` 的 `wl_crm_user`。
- 登入驗證：查 `crm_users`（`account`+`pin`+`active`），命中才登入。
- **權限視角**：
  - `admin`：導覽列出現「視角切換」下拉，可模擬各部門視角（`adminView`）。
  - 一般員工：固定看自己部門（無切換），導覽列顯示 `🏢 部門`。
  - 兩者皆顯示 `👤 姓名` 與「登出」鈕。
  - `currentRole` 為衍生值：admin 時取 `adminView`，否則取使用者 role。
- **姓名背景浮水印**：`Watermark` 於 CRM 詳情頁平鋪登入者姓名（`position:fixed`、`opacity:0.05`、`pointer-events:none`）— 責任歸屬 / 防截圖，不擋操作。
- **案件異動紀錄**（寫入操作者姓名）：
  - 報修：派工、結案、完工 → 寫入 `history`
  - 安裝：建單、排程、到場、施工、無法安裝、完工驗收 →（集中於 `setStatus`）寫入 `history`
  - 客訴：沿用既有 `timeline`，補上 `by`；客訴建立 `createdBy` 改為登入者
  - `xxxToDb` / `dbToXxx` 已補 `history` 欄位以同步至 Supabase
