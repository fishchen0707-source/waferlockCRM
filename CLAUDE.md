# WAFERLOCK CRM — 協作入口

## 🚨 第一守則：動手前先讀版本紀錄

**在修改本專案任何檔案之前，必須先讀完 [`版本紀錄.md`](版本紀錄.md)。**
它記錄「目前做到哪、還缺什麼、誰在做什麼」，是專案進度的單一真實來源。

改完之後，**務必回到 `版本紀錄.md` 的「未發布」區補一筆**（日期／做了什麼／影響範圍／你的名字），再 commit。

## 文件導覽

- [`版本紀錄.md`](版本紀錄.md) — 版本歷程、目前進度、待辦（**每次都先看這個**）
- [`專案總覽.md`](專案總覽.md) — 系統架構、檔案結構、資料表、元件說明

## 專案速覽

- 主檔：`waferlock_crm.html`（單一 HTML，React 18 CDN + Babel Standalone，無建置工具）
- 後端：Supabase（URL/anon key 硬編碼於前端；RLS 目前全開，僅前端閘門）
- 部署：push 到 `main` → GitHub Pages 自動部署正式環境
- 本機預覽：`啟動伺服器.bat`（python http.server 8765）

## 改動規範（沿用既有編程契約）

- 外科手術式修改：只改必要處，不順手重構無關程式。
- 約定優先：CSS class 用 kebab-case、JS 函式用 camelCase、沿用現有錯誤處理與資料同步模式（`xxxToDb`/`dbToXxx`）。
- 顯性失敗：不確定是否成功就明說，不靜默跳過。
- commit 訊息用繁體中文描述「做了什麼」。
- 不硬編碼帳號/密碼/金鑰到程式（anon key 例外，但搭配全開 RLS 屬已知風險）。
